Unicef Project Revision and Reflection

Throughout the project, I made revisions based on peer feedback and personal reflection to improve my HTML code’s coherence and visual appeal. One major change was replacing plain text headers with the official UNICEF logo. This helped the site look more professional. I also updated the color scheme to match UNICEF’s blue, which created consistency across all pages.

On the "Inspiring Stories" page, I made several adjustments. I changed the heading size and color, and added a horizontal bar under the title to better match the layout of the rest of the site. These small but important changes brought visual unity to the overall design. I also revised the article layout by moving images to the side and aligning the text to the right, improving readability. Captions were updated and standardized so each page followed a coherent style. Lastly, I refined all buttons by giving them rounded corners and adjusting their color to fit the site’s theme, making the site feel more modern and interactive.

The feedback I received from peers was extremely helpful. For example, someone pointed out that my buttons looked too generic, which led me to restyle them with round corners and a different color. Looking back, I believe I did well with applying feedback and paying attention to design consistency. If I could start over, I would spend more time planning the structure of each page before coding. I believe I focused too much on the content as I was building my HTML structure, leading me to go back and fix it later. One challenge I encountered during the project was with my CSS. I realized that because I didn’t structure my HTML properly in the early stages, it made applying styles more difficult and disorganized. In my final version, I had to go back and clean up my HTML multiple times. This taught me how important it is to build clean, semantic HTML as a foundation for efficient styling.

I learned to think creatively when building a webpage and gained a better understanding of the importance of users. I originally took this class as an elective, thinking it would just be a fun way to learn about web development, but it exceeded my expectations and turned out to be one of the most valuable classes this semester.

Thoughts on Progressive Enhancement

Progressive enhancement is a web development approach that focuses on the content. Through this method, we begin by forming a solid foundation and layering off of that. A Peanut M&M analogy is a good example of this approach. The peanut is the core content written in semantic HTML, which is usable by all browsers. The chocolate coating represents CSS, which adds layers of visual styling. And the hard candy shell is JavaScript, which adds interactivity. This approach allows every user to access and use the website regardless of the device or browser, the opposite of the graceful degradation approach. The graceful degradation approach was common in the early days of web development, and it focuses on building websites for the most advanced browsers and scaling back to older browsers. This approach does not support the vision of a universally accessible web.

Progressive enhancement is powerful because it allows everyone to access the basic content and functionality of a web page, while also providing enhanced versions of the page to the users with more advanced browsers.
